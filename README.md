# Прогрессивный метроном — PWA

Прогрессивный метроном для гитары в формате Progressive Web App: устанавливается на телефон/десктоп, работает офлайн, не требует магазина приложений.

## Структура проекта

```
metronome-pwa/
├── index.html              — основное приложение
├── manifest.webmanifest    — манифест PWA
├── sw.js                   — Service Worker (офлайн-кэш)
├── icons/
│   ├── icon-192.png        — иконка 192×192 (Android, любая форма)
│   ├── icon-512.png        — иконка 512×512 (сплэш-экран, любая форма)
│   ├── icon-maskable-192.png  — maskable 192×192 (Android адаптивная)
│   ├── icon-maskable-512.png  — maskable 512×512
│   └── favicon.png         — фавикон 32×32
└── README.md               — этот файл
```

## Развёртывание на GitHub Pages (рекомендуется)

GitHub Pages даёт бесплатный HTTPS, что обязательно для PWA.

1. **Создайте репозиторий** на github.com (например, `metronome` или `progressive-metronome`).
2. **Загрузите содержимое папки** в репозиторий — через веб-интерфейс «Add file → Upload files» или через git:
   ```bash
   cd metronome-pwa
   git init
   git add .
   git commit -m "Initial PWA"
   git branch -M main
   git remote add origin https://github.com/<user>/<repo>.git
   git push -u origin main
   ```
3. **Включите Pages**: Settings → Pages → Source: `main` → `/ (root)` → Save.
4. Через 1–2 минуты приложение будет доступно по адресу `https://<user>.github.io/<repo>/`.
5. Откройте этот URL в Chrome на телефоне или десктопе — в адресной строке появится значок установки приложения, либо в шапке появится кнопка «Установить».

После установки приложение запускается с главного экрана как обычное мобильное и работает офлайн.

## Развёртывание на Netlify

Ещё проще: бесплатно, без git.

1. Зайдите на [app.netlify.com/drop](https://app.netlify.com/drop).
2. Перетащите папку `metronome-pwa` целиком в окно браузера.
3. Получите случайный URL вида `https://random-name.netlify.app` — его можно сразу открывать на телефоне.

Для постоянного имени бесплатно настраивается через Netlify-аккаунт: «Domain settings → Change site name».

## Локальный запуск (для разработки и тестирования)

PWA не работает по `file://` — нужен HTTP-сервер. Самый быстрый вариант — Python:

```bash
cd путь/к/metronome-pwa
python -m http.server 8080
```

Откройте `http://localhost:8080` в Chrome. Service Worker зарегистрируется (localhost считается «безопасным контекстом»), и можно проверить установку через DevTools → Application.

Альтернативы:

- Node.js: `npx serve` или `npx http-server`.
- VS Code: расширение **Live Server**.
- Любой другой статический сервер.

## Установка на устройства

### Android (Chrome / Edge / Brave)
1. Откройте сайт.
2. В меню браузера выберите «Установить приложение» / «Добавить на главный экран».
3. Иконка появляется на главном экране. Запускается в standalone-режиме без адресной строки.

### iOS (Safari)
1. Откройте сайт в Safari (другие браузеры на iOS не поддерживают установку).
2. Кнопка «Поделиться» → «На экран Домой».
3. iOS не поддерживает кнопку `beforeinstallprompt`, поэтому установка только через меню Share.

### Десктоп (Chrome / Edge на Windows, macOS, Linux)
1. В правой части адресной строки появится значок установки (плюсик в круге).
2. Либо кнопка «Установить» в шапке самого приложения.
3. После установки запускается из меню «Пуск» / Launchpad как отдельное окно.

## Офлайн-режим

При первом успешном запуске Service Worker кэширует все файлы (HTML, JS, иконки). После этого приложение полностью работает без интернета. Голосовой отсчёт через `SpeechSynthesis` использует системные TTS-голоса, которые тоже доступны офлайн.

В шапке приложения после регистрации SW появляется бейдж «● Готов к офлайн».

## Обновление приложения

1. Внесите изменения в `index.html` или другие файлы.
2. **Поднимите версию кэша** в `sw.js`, например с `metronome-v1.0.0` на `metronome-v1.0.1`. Это критично — старый кэш отдаёт старую версию.
3. Залейте изменения на хостинг.
4. При следующем визите пользователя SW обнаружит обновление и покажет тост «Доступна новая версия» с кнопкой «Обновить» — нажатие перезагружает приложение с новым кэшем.

## Возможности приложения

- Точное время благодаря Web Audio API (lookahead-планировщик).
- Прогрессивный разгон по ударам с учётом тактов.
- Размер такта: без акцента, 2/4, 3/4, 4/4, 6/8, 8/8.
- Четыре тембра клика, регулировка громкости, голосовой отсчёт.
- Пресеты в `localStorage` (3 встроенных + пользовательские).
- Установка как нативное приложение, офлайн-режим.
- Адаптивный glassmorphism-дизайн под мобильные и десктоп.

## Если что-то не работает

**Кнопка «Установить» не появляется**
- Откройте через HTTPS или localhost (не file://).
- Используйте Chrome/Edge на Android или десктопе. iOS Safari требует ручной установки через Share.
- Откройте DevTools → Application → Manifest и убедитесь, что нет ошибок.

**Service Worker не регистрируется**
- DevTools → Application → Service Workers — проверьте статус и ошибки в консоли.
- Убедитесь, что путь `sw.js` доступен (возвращает 200 при прямом открытии `https://.../sw.js`).
- При локальной разработке очистите кэш через DevTools → Application → Storage → Clear site data.

**Звук не воспроизводится после установки**
- Web Audio требует первого жеста пользователя — нажмите «Старт» один раз. На некоторых iOS-устройствах нужно сначала включить системный звук (не silent mode).

**Иконка показывается некрасиво в углу на Android**
- Это нормально для не-maskable иконок: некоторые лаунчеры обрезают по форме. У нас есть `purpose: "maskable"` версии, которые корректно адаптируются под круг/квадрат.
